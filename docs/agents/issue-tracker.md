# Issue tracker

## Primary tracker

This repository uses **GitHub Issues** as the primary issue tracker.

- Create and manage issues with the `gh` CLI.
- Skills that create or triage issues should target GitHub first.

## Local workflow note

This repo also uses **bd (beads)** locally for issue/task tracking workflows.

- `bd` can be used for local/offline planning and execution.
- When work must be reflected in the shared tracker, mirror relevant outcomes to GitHub Issues.

## Operational guidance for agents

1. Prefer GitHub Issues for shared, long-lived tracking.
2. Respect existing local `bd` workflows in this repo.
3. Do not invent a second label/state system beyond configured triage labels.
